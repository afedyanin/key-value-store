# Продвинутая сериализация с генераторами кода

## Цель

- Применить на практике метапрограммирование на этапе компиляции для создания собственного высокопроизводительного компонента.
- Заменить медленную, reflection-based сериализацию на сгенерированный код, чтобы полностью устранить runtime-overhead.
- Получить объективное доказательство преимуществ генерации кода с помощью профессиональных инструментов для бенчмаркинга.
- Освоить одну из самых современных и востребованных техник в арсенале .NET-эксперта.

## Результат

Студент создаст полноценный Source Generator, который автоматически генерирует код для бинарной сериализации. 
Также будет создан BenchmarkDotNet тест, наглядно демонстрирующий, что сгенерированный код на порядки превосходит по производительности стандартный System.Text.Json.

## Зачем это нужно

Это задание — квинтэссенция экспертного подхода к производительности. 
Оно учит не просто использовать готовые инструменты, а создавать свои, когда стандартные решения становятся узким местом. 
Умение писать генераторы кода позволяет решать проблемы производительности на фундаментальном уровне, еще до запуска приложения, и является отличительной чертой разработчика высочайшего класса.


## Описание/Пошаговая инструкция выполнения домашнего задания

### Пункт 1. Создание проекта для Source Generator

Создайте в своем решении новый проект типа "C# Class Library". 
Настройте его так, чтобы он функционировал как Source Generator (подключив необходимые NuGet-пакеты, такие как Microsoft.CodeAnalysis.Analyzers и Microsoft.CodeAnalysis.CSharp). 
Создайте класс, реализующий интерфейс IIncrementalGenerator.

### Пункт 2. Создание атрибута-маркера

В проекте генератора создайте простой public class GenerateBinarySerializerAttribute : Attribute. 
Этот атрибут будет использоваться для пометки DTO-классов, для которых нужно сгенерировать код сериализации.

### Пункт 3. Реализация логики генератора

Реализуйте логику IIncrementalGenerator. 
Он должен находить в коде все классы, помеченные атрибутом [GenerateBinarySerializer]. 
Для каждого такого класса генератор должен анализировать его публичные свойства (поддерживая базовые типы, такие как int, string, DateTime) и создавать partial class с методом SerializeToBinary(Stream stream). 
Этот метод должен напрямую писать байты свойств в поток (например, используя BinaryWriter).

### Пункт 4. Интеграция генератора и использование сгенерированного кода

Подключите ваш проект-генератор к основному проекту сервера. 
Пометьте ваш DTO-класс UserProfile атрибутом [GenerateBinarySerializer]. 
Модифицируйте метод Set в SimpleStore так, чтобы он теперь вызывал новый, сгенерированный метод SerializeToBinary вместо JsonSerializer.Serialize.

### Пункт 5. Сравнение производительности с помощью BenchmarkDotNet

В проекте для нагрузочных тестов (или в новом) создайте класс-бенчмарк с помощью BenchmarkDotNet. 
В нем создайте два метода: один будет сериализовать объект UserProfile с помощью System.Text.Json, а второй — с помощью вашего нового, сгенерированного метода. 
Запустите бенчмарк и проанализируйте результаты, обратив внимание на колонки Mean и Allocated.


## Критерии оценки

- Создан и корректно настроен проект Source Generator, включая класс, реализующий IIncrementalGenerator, и атрибут-маркер - 1 балл
- Генератор корректно находит помеченные атрибутом классы и анализирует их публичные свойства - 3 балла
- Генератор успешно создает partial class с методом для бинарной сериализации, который компилируется без ошибок - 2 балла
- Сгенерированный метод успешно используется в классе SimpleStore вместо стандартного сериализатора - 2 балла
- Написан и запущен BenchmarkDotNet тест, который наглядно доказывает превосходство сгенерированного кода над System.Text.Json по скорости и количеству аллокаций - 2 балла
