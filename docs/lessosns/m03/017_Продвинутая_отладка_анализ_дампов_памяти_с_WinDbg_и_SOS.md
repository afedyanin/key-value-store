# Продвинутая отладка: анализ дампов памяти с WinDbg и SOS

## Цели занятия

- анализировать дампы памяти для диагностики критических сбоев (crash) и зависаний (hang);
- использовать WinDbg и расширение SOS для инспекции состояния управляемой кучи и потоков;
- диагностировать сложные проблемы, такие как дедлоки и повреждение памяти, на низком уровне;
- выполнять post-mortem отладку приложений, работающих в production-среде.

## Краткое содержание

- WinDbg и расширение SOS (Son of Strike) для отладки .NET-приложений;
- анализ crash-дампов: чтение стека вызовов, просмотр исключений;
- анализ hang-дампов: диагностика дедлоков (!dlk), анализ состояния потоков (!threads);
- инспекция управляемой кучи: поиск объектов (!dumpheap), просмотр их содержимого (!do), анализ путей к корням (!gcroot);
- сравнение с высокоуровневыми профилировщиками: когда и зачем нужен WinDbg.

## Результаты

- проанализированный дамп памяти приложения с дедлоком с помощью WinDbg и SOS для выявления причины зависания.
