# Нагрузочное тестирование: проверка вашего кода на прочность

## Цели занятия

- планировать и проводить нагрузочное тестирование для сетевых сервисов;
- использовать NBomber для генерации нагрузки;- анализировать результаты тестов: RPS (запросы в секунду), время отклика (latency), количество ошибок;
- определять максимальную пропускную способность и точки отказа приложения.

## Краткое содержание

- виды тестирования производительности: нагрузочное, стрессовое, на выносливость (soak test);
- введение в NBomber: философия "тесты как код" для .NET-разработчиков;
- практика: написание сценария для тестирования нашего TCP-кэша;
- создание шагов (Step) для отправки кастомных TCP-команд
- построение сценария (Scenario) с фазами разогрева (warm-up) и основной нагрузки;
- настройка симуляции нагрузки (Simulation.Inject, Simulation.KeepConstant);
- запуск теста и сбор ключевых метрик: RPS, p95/p99 latency, количество ошибок;
- корреляция результатов нагрузочного теста с данными из OpenTelemetry и dotnet-counters для комплексного поиска узких мест.

## Результаты

- сценарий нагрузочного теста для курсового проекта, написанный на NBomber, и отчет с результатами анализа производительности (RPS, latency).
