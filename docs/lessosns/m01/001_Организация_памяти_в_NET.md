# Организация памяти в .NET: Stack, Heap, LOH и GC

## Цели занятия

- анализировать использование памяти приложением для выявления потенциальных проблем;
- определять связанные с памятью (боксинг, фрагментация LOH) причины проблем с производительностью;
- обосновывать выбор между class и struct с точки зрения паттернов аллокации.

## Краткое содержание

- стек (Stack) и Куча (Managed Heap): модели хранения и жизненный цикл объектов;
- типы-значения (Value Types) и ссылочные типы (Reference Types);
- боксинг (boxing) и анбоксинг (unboxing): скрытые аллокации и их влияние на производительность;
- куча больших объектов (Large Object Heap, LOH) и проблемы ее фрагментации;
- основы работы Сборщика мусора (Garbage Collector, GC): поколения, базовые принципы.
