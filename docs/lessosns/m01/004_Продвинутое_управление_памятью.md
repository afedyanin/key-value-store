# Продвинутое управление памятью: ArrayPool<T> и MemoryPool<T>

## Цели занятия

- использовать ArrayPool<T> для переиспользования массивов и снижения нагрузки на GC;
- применять паттерн "пул объектов" для управления жизненным циклом часто используемых ресурсов;
- проектировать системы (например, сетевые серверы), устойчивые к пиковым нагрузкам за счет пулинга буферов.

## Краткое содержание

- паттерн ""Пул объектов"" (Object Pooling);
- System.Buffers.ArrayPool<T>: получение и возврат массивов в общий пул;
- System.Buffers.MemoryPool<T>: более абстрактный механизм для работы с блоками памяти;
- правила использования пулов: своевременный возврат ресурсов, обработка IDisposable;
- сценарии применения: обработка потоков данных, работа с изображениями, сетевые протоколы.

## Результаты

код, использующий ArrayPool<T> для переиспользования буферов и сравнения количества аллокаций с подходом без пулинга.
