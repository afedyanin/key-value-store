# Сборщик мусора (GC) под микроскопом: поколения, режимы и тюнинг

## Цели занятия

- настраивать режим работы GC (Server/Workstation) для оптимизации под конкретный тип нагрузки;
- использовать WeakReference<T> для создания кэшей, устойчивых к давлению памяти;
- диагностировать работу GC в реальном времени с помощью dotnet-counters для контроля производительности.

## Краткое содержание

- режимы GC: Server vs Workstation;
- настройка GC через .csproj и runtimeconfig.json WeakReference<T> и WeakHashMap: паттерны кэширования;
- управление задержками: GCSettings.LatencyMode и GC.TryStartNoGCRegion;
- мониторинг GC с помощью dotnet-counters.

## Результаты

- приложение, сконфигурированное для режима Server GC;
- результаты анализа поведения GC через dotnet-counters.
