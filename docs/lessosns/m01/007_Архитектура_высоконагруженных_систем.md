# Архитектура высоконагруженных систем: System Design на примере In-Memory кэша

## Цели занятия

- декомпозировать сложную задачу на логические компоненты и сервисы;
- проектировать API и протокол взаимодействия для сетевого сервиса;
- выбирать подходящие технологии и структуры данных на основе нефункциональных требований (производительность, масштабируемость);
- выявлять потенциальные узкие места и точки отказа на этапе проектирования.

## Краткое содержание

- введение в System Design: функциональные и нефункциональные требования;
- Проектирование High-Level архитектуры: Сетевой уровень, Протокольный уровень, Конвейер обработки, Ядро хранилища;
- проектирование простого текстового протокола (аналог RESP) Deep Dive в компоненты: выбор между ConcurrentDictionary и Dictionary + ReaderWriterLockSlim;
- Оценка и выявление узких мест: анализ потенциальных точек отказа и узких мест производительности.

## Результаты
- архитектурная схема и краткое техническое описание основных компонентов курсового проекта.
