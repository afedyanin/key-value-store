# Профилирование CPU: BenchmarkDotNet и dotnet-trace

## Цели занятия

- измерять производительность методов с помощью BenchmarkDotNet для принятия обоснованных решений по оптимизации;
- использовать dotnet-trace для сбора трассировок и анализа производительности приложения в целом;
- находить "горячие" точки (hot paths) в коде, которые потребляют больше всего процессорного времени.

## Краткое содержание

- библиотека BenchmarkDotNet: написание и запуск бенчмарков;
- утилита dotnet-trace: сбор и анализ трассировок CPU;
- визуализация трассировок в PerfView и Visual Studio;
- понятие Flame Graphs для анализа производительности.

## Результаты

- бенчмарк, созданный с помощью BenchmarkDotNet, и файл трассировки приложения, полученный с помощью dotnet-trace.
