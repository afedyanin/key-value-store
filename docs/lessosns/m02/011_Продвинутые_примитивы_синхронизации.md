# Продвинутые примитивы синхронизации: SemaphoreSlim, ReaderWriterLockSlim, ManualResetEventSlim

## Цели занятия

- применять SemaphoreSlim для ограничения количества одновременных операций;
- использовать ReaderWriterLockSlim для оптимизации доступа к данным в сценариях с преобладанием чтения;
- организовывать "ручное" управление доступом с помощью ManualResetEventSlim.

## Краткое содержание

- semaphoreSlim: асинхронный семафор;
- ReaderWriterLockSlim: реализация паттерна "Читатели-писатели";
- ManualResetEventSlim: управление состоянием "сигнал/не сигнал" для потоков;
- сравнение производительности примитивов в разных сценариях.

## Результаты

- код, использующий ReaderWriterLockSlim, и сравнительный анализ производительности с lock.
