# Профилирование памяти: dotnet-dump и dotMemory

## Цели занятия

- создавать и анализировать дампы памяти для диагностики проблем;
- находить утечки памяти, анализируя пути к корням объектов (GC roots);
- выявлять источники избыточных аллокаций, которые приводят к частым сборкам мусора.

## Краткое содержание

- утилита dotnet-dump: создание дампов памяти работающего процесса;
- утилита dotnet-gcdump: сбор информации о куче без остановки процесса;
- анализ дампов в Visual Studio и JetBrains dotMemory;
- поиск утечек памяти и анализ доминирующих объектов.

## Результаты

- дамп памяти приложения, созданный с помощью dotnet-dump, и результаты его анализа.
